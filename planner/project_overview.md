# 프로젝트 전체 개요

## 1. 프로젝트 소개

| 항목            | 내용                                                                                                    |
| --------------- | ------------------------------------------------------------------------------------------------------- |
| **프로젝트명**  | 마레픽 (My Recipe Pick)                                                                                 |
| **한줄 설명**   | 유튜브 요리 영상을 AI로 정리하고, 요리할 때마다 나만의 메모를 쌓아, 쓸수록 나를 알아가는 개인 레시피 북 |
| **목표 사용자** | 유튜브 요리 영상을 보고 따라하는 일반 가정 요리사 (IT 비숙련자 포함)                                    |
| **핵심 가치**   | "쓰면 쓸수록 나를 알아가는 레시피 북"                                                                   |

### 프로젝트 본질

1. AI가 유튜브 레시피를 깔끔하게 정리해준다 (1단계)
2. 사용자가 요리하면서 재료별/단계별로 자기만의 메모를 남긴다 (2단계)
3. 시간이 지나면 나만의 입맛 패턴이 쌓인다 -- "단맛은 조청 선호", "매운 건 약하게" (3단계)
4. 새 레시피를 볼 때 AI가 내 입맛에 맞게 자동 제안한다 -- "설탕 대신 조청 쓰시면 좋을 것 같아요" (3단계)

## 3. 3단계 로드맵 및 진행 현황

### 1단계: AI 레시피 정리 -- 진행률 **95%**

핵심 파이프라인(YouTube URL -> 오디오 추출 -> Gemini 분석 -> 구조화 JSON -> DB 캐싱)이 완성되어 실제 작동 가능한 상태.

| 기능                                                                  | 상태                                                                        |
| --------------------------------------------------------------------- | --------------------------------------------------------------------------- |
| YouTube URL에서 video_id 추출 (yt-dlp + regex fallback)               | 완료                                                                        |
| 오디오 다운로드 (yt-dlp + FFmpeg -> MP3)                              | 완료                                                                        |
| Gemini 멀티모달 분석 (오디오 + 메타데이터 -> 구조화 JSON)             | 완료                                                                        |
| Pydantic v2 모델 검증 (Recipe, Ingredient, RecipeStep, FlavorProfile) | 완료                                                                        |
| video_id 기반 DB 캐싱 (중복 분석 방지)                                | 완료                                                                        |
| 비-레시피 영상 필터링 (is_recipe 플래그)                              | 완료                                                                        |
| 에러 핸들링 (구조화된 ErrorResponse)                                  | 완료                                                                        |
| analysis_logs 기록                                                    | 완료                                                                        |
| Gemini API 재시도 (tenacity, 3회)                                     | 완료                                                                        |
| **카테고리 자동 분류 (AI 제안)**                                      | **완료** — Gemini 프롬프트에 15종 고정 목록 적용, `category` 필드 전체 연동 |
| 프론트엔드 기본 UI (랜딩/로딩/결과 3화면)                             | 완료 (새 디자인으로 재구현 예정)                                            |
| 프론트엔드 에러 상세 처리                                             | **미흡**                                                                    |
| 프론트엔드 반응형 디자인                                              | **미구현**                                                                  |

### 2단계: 나만의 레시피 변형 노트 -- 진행률 **40%**

사용자가 요리하면서 재료별/단계별로 개인 메모를 남기고, 요리 후기를 쌓아가는 단계.

| 기능                                       | 상태           | 설명                                         |
| ------------------------------------------ | -------------- | -------------------------------------------- |
| 레시피북 저장 (POST /collections)          | 완료           | 기본 저장 가능                               |
| 레시피북 조회 (GET /collections/{user_id}) | 완료           | recipes JOIN 반환                            |
| 레시피북 삭제 (DELETE /collections/{id})   | **완료**       | 인라인 confirm + 목록으로 이동               |
| 내 메모 수정 (PATCH /collections/{id})     | **완료**       | textarea 인라인 편집 + 토스트 알림           |
| 카테고리 필터 탭 (프론트엔드)              | **완료**       | 클라이언트 사이드 필터링                     |
| **카테고리 AI 제안 + 사용자 확인**         | **부분 구현**  | AI가 자동 분류 (확인/수정 UI 미구현)         |
| **재료별 인라인 메모**                     | **미구현**     | "설탕 -> 물엿으로 대체하니 더 촉촉함"        |
| **단계별 인라인 메모**                     | **미구현**     | "고춧가루 2큰술 -> 1.5큰술이 우리 입맛에 딱" |
| **요리 후기 (날짜별 쌓임)**                | **미구현**     | 요리할 때마다 후기 추가                      |
| 사용자 인증 (Supabase Auth JWT)            | **스켈레톤만** |                                              |

### 3단계: 입맛 패턴 분석 및 AI 자동 제안 -- 진행률 0%

2단계에서 쌓인 변형 메모 데이터를 분석하여 사용자의 입맛 패턴을 파악하고, 새 레시피에 자동으로 맞춤 제안을 하는 단계.

| 기능                         | 상태       | 설명                                                                         |
| ---------------------------- | ---------- | ---------------------------------------------------------------------------- |
| **입맛 패턴 추출**           | **미구현** | 변형 메모에서 선호 패턴 자동 분석 (예: "단맛 -> 조청 선호", "매운맛 약하게") |
| **재료 대체 제안**           | **미구현** | "설탕 대신 조청 쓰시면 좋을 것 같아요"                                       |
| **양 조절 제안**             | **미구현** | "매운맛 3 -> 2로 줄이시면 입맛에 맞을 거예요"                                |
| **맛 프로필 매칭**           | **미구현** | 레시피 맛 지표 vs 사용자 선호 맛 비교                                        |
| taste_profiles 테이블 및 API | **미구현** | 사용자 입맛 프로필 저장                                                      |
| 냉장고 연동 / 쇼핑 리스트    | **미구현** | 확장 기능                                                                    |

## 4. 핵심 기능 요약

### 구현 완료

1. **AI 레시피 정리**: YouTube URL -> Gemini 2.5 Flash -> 재료/조리법/맛지표/팁 JSON
2. **스마트 캐싱**: video_id 기반 DB 캐시로 중복 분석 방지 (즉시 반환)
3. **지능형 필터링**: 먹방/뉴스/음악 등 비-레시피 영상 자동 거부
4. **구조화 데이터**: 재료(카테고리별), 조리법(타이머 포함), 맛 5축 지표
5. **프론트엔드 3화면**: 랜딩(URL 입력) -> 로딩(정리 중) -> 결과(레시피 표시)
6. **레시피북 저장**: 개인 메모 + 재료 가감 정보 저장

### 다음 구현 (P0)

1. 프론트엔드 새 디자인("레시피 북" 컨셉)으로 재구현
2. 레시피북 조회 API + 프론트엔드 연동
3. 카테고리 AI 제안 (Gemini 프롬프트 고정 목록)
4. 에러 처리 고도화
5. 반응형 디자인

### 향후 구현 (P1~P2)

1. 사용자 인증 (Supabase Auth)
2. 재료별/단계별 인라인 메모 (변형 노트)
3. 요리 후기 시스템
4. 입맛 패턴 분석 + AI 자동 제안
