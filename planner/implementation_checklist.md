# 구현 가능 여부 분류 체크리스트

`frontend_design_concept.md` 기준으로 각 항목을 세 가지로 분류한다.

- ✅ **완료** — 현재 코드에 이미 구현됨
- 🔧 **코드로 구현 가능** — 디자이너 없이 CSS/JS/컴포넌트 작업으로 가능
- 🎨 **디자이너 에셋 필요** — 디자인 작업물(이미지, 로고, 일러스트)이 있어야 구현 가능
- ⚙️ **백엔드 작업 선행 필요** — API나 DB가 없어서 프론트만으론 완성 불가

---

## 1. 페이지 구성

| 항목 | 상태 | 비고 |
|------|------|------|
| 홈 페이지 `/` | ✅ 완료 | URL 입력 + 결과 인라인 표시 |
| 로딩 화면 (인라인 진행 상태) | ✅ 완료 | 썸네일 + 단계 텍스트 |
| 레시피 결과 페이지 | ✅ 완료 | 단일 칼럼, max-width 720px |
| 내 레시피북 `/library` | ✅ 완료 | 목록 표시 |
| 저장된 레시피 상세 `/library/[id]` | ✅ 완료 | 내 메모 포함 |
| 에러 페이지 | ✅ 완료 | 404/500 분기 |
| 로그인 모달 (별도 페이지 없이) | ⚙️ Auth 선행 필요 | Supabase Auth 구현 후 |

## 9. 진짜 디자이너가 필요한 것 (3가지뿐)

| 항목 | 이유 | 임시 대체 방법 |
|------|------|----------------|
| **로고 / 워드마크** | "Recipe Book" 브랜드 로고 디자인 | 현재 텍스트 로고로 유지 가능 |
| **실제 종이 텍스처 이미지** | 고품질 종이 질감 (PNG/JPG) | CSS SVG 노이즈 필터로 90% 대체 가능 |
| **앱 고양이 캐릭터** | Flutter 앱 전용, 일러스트 작업 | 웹에는 불필요 (디자인 원칙상 배제) |

> **결론**: 웹 프론트엔드 기준으로 디자이너 없이 구현 불가능한 항목은 사실상 없다.
> 로고와 텍스처는 임시 대체가 가능하므로, 런칭 후 브랜딩 리뉴얼 단계에서 디자이너를 붙이면 된다.
