# 프론트엔드 디자인 컨셉: "나만의 레시피 북"

## 1. 디자인 컨셉 정의

### 1.1 핵심 컨셉

**"나만의 레시피 북"** -- 잘 정리된 요리책처럼, AI가 유튜브 영상 속 레시피를 깔끔하게 정리해주는 개인 요리책.

| 항목 | 정의 |
|------|------|
| **메타포** | 요리책 / 레시피 북 / 정리된 요리 노트 |
| **톤** | 따뜻하고 정돈된, 한 권의 요리책을 넘기는 느낌 |
| **타겟 감정** | "내 요리책에 또 한 페이지가 추가됐다" |
| **타겟 사용자** | IT에 익숙하지 않은 일반 가정 요리사 (40~60대 포함) |

### 1.2 디자인 원칙

1. **극도의 단순함**: URL 붙여넣기 -> 버튼 하나 -> 결과. 그 이상의 복잡함은 없다.
2. **스크롤만으로 완결**: 결과 화면은 탭, 토글, 복잡한 인터랙션 없이 위에서 아래로 스크롤하면 모든 정보를 확인할 수 있다.
3. **캐릭터 배제 (웹)**: 고양이 캐릭터는 앱 전용. 웹에서는 타이포그래피와 레이아웃으로 개성을 표현한다.
4. **요리책 한 페이지**: 결과 페이지는 펼쳐진 요리책의 한 페이지를 보는 듯한 적정 너비와 여백을 유지한다.
5. **쉬운 말**: "AI 분석" 대신 "레시피 정리 중", "Extract" 대신 "정리하기" 등 기술 용어를 쓰지 않는다.
6. **반응형 기본**: 모바일 웹에서도 동일한 경험을 제공한다.

### 1.3 용어 통일

기술적이거나 어려운 표현을 사용자 친화적으로 변환한다.

| 기존 표현 | 변경 표현 |
|-----------|-----------|
| AI 분석 | 레시피 정리 |
| 분석 중 | 레시피 정리하는 중... |
| 추출 | 정리 |
| 저장하기 | 내 레시피북에 추가 |
| 보관함 / 나의 주방 | 내 레시피북 |
| 컬렉션 | 레시피북 |
| 맛 프로필 | 맛 한눈에 보기 |
| FlavorProfile | 맛 특징 |
| 에러 발생 | 문제가 생겼어요 |

### 1.4 기존 디자인과의 차이점

| 항목 | 기존 (캐릭터 컨셉) | 새로운 (레시피 북 컨셉) |
|------|---------------------|--------------------------|
| 히어로 영역 | 고양이 캐릭터 일러스트 | 타이포그래피 + 검색 입력 중심 |
| 로딩 화면 | 별도 페이지 + 캐릭터 | 인라인 프로그레스 (쉬운 단계 텍스트) |
| 결과 페이지 | 2단 Split + 썸네일 | 단일 칼럼 요리책 페이지 (영상 없음) |
| 보관함 | 이미지 그리드 | 요리책 목차 느낌 리스트 |
| 저장 표현 | "저장하기" | "내 레시피북에 추가" |
| 전체 분위기 | 귀여움, 캐주얼 | 실용적, 요리책의 따뜻한 정돈감 |

## 2. 페이지 구성안

### 2.1 페이지 목록

| 페이지 | 라우트 | 역할 |
|--------|--------|------|
| **홈** | `/` | URL 입력 + 레시피 정리 요청의 진입점 |
| **레시피 페이지** | `/recipe/{video_id}` | 정리된 레시피 상세 (요리책 한 페이지) |
| **내 레시피북** | `/library` | 저장된 레시피 목록 (요리책 목차) |
| **저장된 레시피 상세** | `/library/{id}` | 보관된 레시피 + 내 메모 |

> 로그인/회원가입은 모달로 처리 (별도 페이지 불필요).

### 2.2 페이지별 상세

#### 홈 페이지 (`/`)

단 하나의 목적: 유튜브 URL을 붙여넣고 버튼을 누르는 것.

```

│                                           │

│         깔끔하게 정리해드려요              │
│                                           │
│   ┌───────────────────────────────────┐   │
│   │                                   │   │
│   │  유튜브 링크를 여기에 붙여넣으세요  │   │
│   │                                   │   │
│   │                   [ 레시피 정리하기 ]│   │
│   └───────────────────────────────────┘   │
│                                           │
│                                           │
│         사용법: 링크 붙여넣기 -> 버튼 누르기│
│         그러면 재료와 만드는 법이 정리돼요  │
│                                           │
│                                           │
│                                           │
└───────────────────────────────────────────┘
```

**핵심 설계 의도**:
- 화면 중앙에 URL 입력과 버튼만 강조
- 설명 텍스트도 "링크 붙여넣기 -> 버튼 누르기"처럼 극도로 단순하게
- 기능 소개 카드 등 부가 정보는 최소화하거나 스크롤 아래에 배치
- 여백을 넉넉하게 사용하여 시선이 분산되지 않도록

#### 레시피 정리 중 (인라인 진행 상태)

별도 페이지 이동 없이, 홈 페이지 내에서 진행 상태를 보여준다. 기술 용어 없이 쉬운 문장으로.

**핵심 설계 의도**:
- 각 단계를 사용자가 이해할 수 있는 쉬운 문장으로
- "오디오 추출" -> "영상을 듣고 있어요"
- "AI 분석 중" -> "재료와 순서를 정리하는 중"
- 프로그레스가 진행되는 느낌을 줘서 기다림을 줄임

- **구분선 스타일**: 요리책의 섹션 구분 느낌 (가는 선 + 섹션명).

## 3. 핵심 사용 흐름

### 3.1 메인 플로우: 레시피 정리 및 저장

```
[홈 페이지]
    |
    |  (1) 유튜브 링크 붙여넣기
    |  (2) "레시피 정리하기" 버튼 누르기
    |
    v
[인라인 진행 표시]
    |
    |  (3) "레시피를 정리하고 있어요..." (30초~1분)
    |      쉬운 문장으로 단계 표시
    |
    v
[레시피 페이지로 자동 이동]  (/recipe/{video_id})
    |
    |  (4) 위에서 아래로 스크롤하며 레시피 확인
    |      제목 -> 맛 -> 재료 -> 만드는 법 -> 꿀팁
    |
    +-- (5a) "내 레시피북에 추가" -> 토스트 알림
    |
    +-- (5b) "새 레시피 정리" -> 홈으로
    |
    +-- (5c) 재료 체크박스 -> 장보기 활용
    |
    +-- (5d) "원본 영상 보기" 링크 -> YouTube 이동
```

### 3.2 보관함 플로우: 내 레시피북 탐색

```
[내 레시피북]  (/library)
    |
    |  (1) 목록에서 레시피 찾기 (카테고리 탭 or 스크롤)
    |
    v
[레시피 카드 누르기]
    |
    |  (2) 저장된 레시피 상세  (/library/{id})
    |
    v
[레시피 내용 확인]
    |
    |  (3) 내 메모 확인/수정
    |  (4) 원본 영상 보기 링크
    |
    +-- (5) "삭제" -> "정말 삭제할까요?" 확인 -> 삭제
```

    |

### 5.2 타이포그래피

가독성 최우선. 요리책의 깔끔한 활자 느낌.

| 용도 | 폰트 | 무게 | 근거 |
|------|------|------|------|
| **제목** | Pretendard | 700 (Bold) | 깔끔한 한글 고딕, 가독성 우수 |
| **본문** | Pretendard | 400 (Regular) | 넉넉한 line-height(1.7)로 읽기 편하게 |
| **수치/시간** | Nunito | 600 (SemiBold) | 재료 수량, 타이머 등 숫자 표현 |
| **꿀팁/메모** | Nanum Myeongjo | 400 | 요리책의 필기 메모 느낌 (제한적 사용) |

> 기존 GmarketSans는 캐릭터 컨셉에 맞는 폰트였으므로 Pretendard로 변경.
> 본문 line-height를 1.7 이상으로 설정하여 레시피 내용 읽기에 편안함 확보.
> 글씨 크기를 전반적으로 키움 (본문 16px -> 17px, 제목 24px -> 28px).

### 5.3 시각 요소

요리책 느낌을 주는 미묘한 장식 요소들:

| 요소 | 적용 위치 | 설명 |
|------|-----------|------|
| **섹션 구분선** | 맛/재료/만드는법 사이 | 가는 선 + 섹션명 (예: "── 재료 ──") |
| **재료 점선** | 재료 이름과 수량 사이 | 요리책의 점선 연결 (`border-bottom: dotted`) |
| **부드러운 그림자** | 레시피 카드 | 종이 느낌 (`box-shadow: 0 2px 8px rgba(0,0,0,0.06)`) |
| **둥근 모서리** | 카드, 버튼 | `border-radius: 12px` |
| **타이머 배지** | 시간 있는 조리 단계 | 둥근 배지 (Terracotta 배경 + 흰 글씨) |
| **꿀팁 카드** | 팁 영역 | Warm Yellow 배경의 부드러운 카드 |
| **북마크 리본** | "레시피북에 추가" 아이콘 | 책갈피 형태 |
| **적정 너비** | 레시피 페이지 전체 | `max-width: 720px` (책 페이지 비율) |

## 6. 반응형 전략

### 6.1 브레이크포인트

| 이름 | 범위 | 대상 |
|------|------|------|
| **Mobile** | ~767px | 스마트폰 |
| **Tablet** | 768px~1023px | 태블릿, 작은 노트북 |
| **Desktop** | 1024px~ | 데스크톱, 큰 노트북 |

### 6.2 레시피 페이지 -- 단일 칼럼이므로 반응형이 자연스러움

레시피 페이지가 `max-width: 720px` 단일 칼럼이므로, 모든 화면 크기에서 동일한 레이아웃을 유지한다. 이것이 "요리책 한 페이지" 컨셉의 큰 장점.

| 요소 | Desktop | Tablet | Mobile |
|------|---------|--------|--------|
| 레시피 카드 너비 | 720px (중앙) | 좌우 패딩 24px | 좌우 패딩 16px |
| 맛 프로필 | 2행 가로 배치 | 2행 가로 배치 | 2행 가로 배치 |
| 재료 목록 | 그대로 | 그대로 | 그대로 |
| 조리 단계 | 그대로 | 그대로 | 그대로 |
| "추가" 버튼 | 상단 + 하단 | 상단 + 하단 | 하단 고정 (sticky) |

### 6.3 페이지별 반응형 동작

#### 홈 페이지
| 요소 | Desktop | Tablet | Mobile |
|------|---------|--------|--------|
| URL 입력 | 최대 600px 중앙 | 좌우 패딩 24px | 좌우 패딩 16px |
| 버튼 | 입력 옆 인라인 | 입력 옆 인라인 | 입력 아래 전체 너비 |

#### 내 레시피북
| 요소 | Desktop | Tablet | Mobile |
|------|---------|--------|--------|
| 카테고리 탭 | 전체 표시 | 전체 표시 | 가로 스크롤 |
| 레시피 목록 | `max-width: 720px` 리스트 | 좌우 패딩 24px | 좌우 패딩 16px |

### 6.4 모바일 사용성 원칙

1. **큰 터치 영역**: 모든 버튼/링크 최소 48px x 48px (IT 비숙련자 고려)
2. **큰 글씨**: 모바일 본문 최소 16px, 제목 22px 이상
3. **하단 고정 버튼**: "내 레시피북에 추가" 버튼을 모바일에서 화면 하단에 고정
4. **스크롤만으로 완결**: 탭이나 펼치기 없이 스크롤로 모든 정보 접근
5. **URL 입력 최적화**: 모바일에서 URL 붙여넣기가 쉽도록 입력 필드를 크게
6. **명확한 피드백**: 버튼 누름, 저장 완료, 에러 등 모든 액션에 즉각적인 시각 피드백
