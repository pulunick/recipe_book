# 🏷️ 레시피북 태그 & 아카이빙 시스템 설계

> 목적: "내 레시피북"을 유튜브 저장 방식보다 더 편하게 — 저장만 하는 게 아니라 **찾고, 분류하고, 내 입맛을 기록하는** 개인 레시피 관리 공간으로 발전시킨다.

---

## 1. 핵심 설계 원칙

| 원칙 | 설명 |
|------|------|
| **분류는 자동, 정리는 자유** | AI가 카테고리(한식/양식 등)는 자동으로 붙여주고, 유저는 원하는 태그를 자유롭게 추가 |
| **즐겨찾기 ≠ 태그** | "내 최애 레시피"는 태그가 아닌 별도 1급 데이터 — 별점 + 만들어본 횟수로 관리 |
| **깊이(Depth)는 2단계** | 1depth = 카테고리(자동), 2depth = 내 태그(수동) — 3단계 이상은 오히려 불편 |
| **모바일 우선** | 태그 추가/삭제는 카드에서 원터치로 완료 |

---

## 2. 기능 구성

### 2.1 태그 2-Depth 구조

```
내 레시피북
├── [1depth] 카테고리 (AI 자동 분류 — 수정 가능)
│   ├── 한식
│   ├── 양식
│   ├── 중식
│   ├── 분식
│   ├── 국/찌개
│   └── 디저트/음료
│
└── [2depth] 내 태그 (유저 자유 생성)
    ├── 손님초대용
    ├── 10분요리
    ├── 다이어트
    ├── 아이반찬
    └── 또 만들고 싶어  ← 예시
```

- **1depth 카테고리**: 레시피 저장 시 AI가 분류한 `category` 값이 자동 적용. 유저가 직접 수정 가능.
- **2depth 내 태그**: 유저가 자유롭게 생성/삭제. 복수 태그 부착 가능. 색상 지정 가능(8색).
- 하나의 레시피에 카테고리 1개 + 내 태그 N개 동시 부착 가능.

### 2.2 즐겨찾기 & 내 별점 시스템

**즐겨찾기(⭐)**
- 레시피 카드 우상단 별 아이콘 원터치 토글.
- 레시피북 상단에 "즐겨찾기" 섹션 고정 노출.
- 유튜브의 "좋아요 재생목록"처럼 빠르게 접근.

**내 별점(1~5)**
- 실제로 만들어본 후 남기는 평가.
- "이번에 만들어봤어요" 버튼 → 별점 입력 모달 → `cooked_count` +1, `last_cooked_at` 업데이트.
- 별점이 있는 레시피는 카드에 별점 뱃지 표시.
- "내 최애 5점짜리" 필터로 바로 모아볼 수 있음.

**만들어본 횟수(🍳)**
- `cooked_count` 누적 카운트.
- 3번 이상 만든 레시피 → "단골 레시피" 뱃지 자동 부착.

## 3. DB 스키마 변경

### 3.1 신규 테이블: `collection_tags` (유저 태그 마스터)

```sql
CREATE TABLE collection_tags (
  id         BIGINT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
  user_id    UUID    REFERENCES users(id) ON DELETE CASCADE,
  name       TEXT    NOT NULL,
  color      TEXT    NOT NULL DEFAULT '#e8ddd4',  -- 태그 색상 (hex)
  created_at TIMESTAMP WITH TIME ZONE DEFAULT NOW(),
  UNIQUE(user_id, name)
);
```

색상 선택지 8가지(기획):
`#f28b82`(레드) `#fbbc04`(옐로우) `#34a853`(그린) `#4285f4`(블루)
`#a8c7fa`(라이트블루) `#e6c9a8`(브라운) `#d3d3d3`(그레이) `#e8ddd4`(기본)

### 3.2 신규 테이블: `collection_tag_items` (컬렉션-태그 연결)

```sql
CREATE TABLE collection_tag_items (
  collection_id BIGINT REFERENCES user_collections(id) ON DELETE CASCADE,
  tag_id        BIGINT REFERENCES collection_tags(id) ON DELETE CASCADE,
  PRIMARY KEY (collection_id, tag_id)
);
```

### 3.3 `user_collections` 테이블 컬럼 추가

```sql
ALTER TABLE user_collections
  ADD COLUMN is_favorite   BOOLEAN   DEFAULT FALSE,
  ADD COLUMN my_rating     SMALLINT  CHECK (my_rating BETWEEN 1 AND 5),  -- NULL = 미평가
  ADD COLUMN cooked_count  INTEGER   DEFAULT 0,
  ADD COLUMN last_cooked_at TIMESTAMP WITH TIME ZONE,
  ADD COLUMN category_override TEXT;  -- AI 자동 분류를 유저가 수정한 경우 덮어쓰기
```

> `category_override`가 NULL이면 `recipes.category`(AI 분류)를 그대로 사용, 값이 있으면 유저가 수정한 값 사용.

### 3.4 완성된 `user_collections` 전체 구조

| 컬럼명 | 타입 | 설명 |
|--------|------|------|
| `id` | bigint PK | 레코드 ID |
| `user_id` | uuid FK | users 참조 |
| `recipe_id` | bigint FK | recipes 참조 |
| `custom_tip` | text | 나만의 메모 |
| `ingredient_adjustments` | jsonb | 제외 재료 |
| `is_favorite` | boolean | 즐겨찾기 여부 |
| `my_rating` | smallint 1-5 | 내 별점 (미평가 시 NULL) |
| `cooked_count` | integer | 만들어본 횟수 |
| `last_cooked_at` | timestamp | 마지막 요리 날짜 |
| `category_override` | text | 카테고리 수동 수정값 |
| `created_at` | timestamp | 저장 일시 |

## 4. API 엔드포인트 설계

### 기존 엔드포인트 확장

| Method | Path | 설명 |
|--------|------|------|
| `GET` | `/collections/{user_id}` | 내 레시피 목록 조회 (필터/정렬 지원) |
| `POST` | `/collections` | 레시피 저장 (기존) |
| `PATCH` | `/collections/{id}` | 메모/재료 수정 (기존 확장) |
| `DELETE` | `/collections/{id}` | 보관함에서 제거 |

### 신규 엔드포인트

| Method | Path | 설명 |
|--------|------|------|
| `PUT` | `/collections/{id}/favorite` | 즐겨찾기 토글 |
| `PUT` | `/collections/{id}/rating` | 별점 설정 (body: `{rating: 1-5}`) |
| `POST` | `/collections/{id}/cooked` | "만들어봤어요" 기록 (body: `{rating?: 1-5}`) |
| `PUT` | `/collections/{id}/category` | 카테고리 수동 변경 |
| `GET` | `/tags/{user_id}` | 내 태그 목록 조회 |
| `POST` | `/tags` | 태그 생성 (body: `{user_id, name, color}`) |
| `DELETE` | `/tags/{id}` | 태그 삭제 (연결된 items 자동 cascade) |
| `PUT` | `/collections/{id}/tags` | 태그 부착/해제 (body: `{tag_ids: [1,2,3]}`) |

### `GET /collections/{user_id}` 쿼리 파라미터

```
?category=한식          # 카테고리 필터
&tag_id=3               # 태그 필터
&is_favorite=true       # 즐겨찾기만
&min_rating=4           # 별점 4이상
&sort=last_cooked       # 정렬: saved_at | last_cooked | rating | cooked_count
&q=된장                 # 제목 검색
```

## 5. 프론트엔드 UX 설계

### 5.1 레시피북 페이지 (`/my-recipes`)

```
┌─────────────────────────────────────────────────┐
│  내 레시피북          [+ 태그 관리]  [검색 🔍]   │
├──────────────┬──────────────────────────────────┤
│  사이드바     │  레시피 카드 그리드               │
│              │                                  │
│  전체 (12)   │  ┌────────┐  ┌────────┐          │
│  ⭐즐겨찾기   │  │[썸네일]│  │[썸네일]│          │
│              │  │ 제목   │  │ 제목   │          │
│  카테고리    │  │한식 ✦  │  │양식    │          │
│  ├ 한식 (5) │  │★★★★☆  │  │       │          │
│  ├ 양식 (3) │  └────────┘  └────────┘          │
│  └ 분식 (2) │                                  │
│              │  ┌────────┐  ┌────────┐          │
│  내 태그     │  │        │  │        │          │
│  ├ 손님초대  │  └────────┘  └────────┘          │
│  ├ 10분요리  │                                  │
│  └ + 추가   │                                  │
└──────────────┴──────────────────────────────────┘
```

**모바일**: 사이드바 → 상단 수평 스크롤 탭으로 전환
```
[전체] [⭐즐겨찾기] [한식] [양식] [손님초대] [+ 추가]
```

### 5.2 레시피 카드 컴포넌트

```
┌──────────────────────────┐
│ [유튜브 썸네일]      ⭐  │  ← 별 아이콘: 즐겨찾기 토글
│                          │
│ 된장찌개 황금 레시피      │
│                          │
│ [한식] [손님초대] [+태그]│  ← 태그 뱃지 + 추가 버튼
│                          │
│ ★★★★☆  🍳 3회 만들어봄  │  ← 별점 + 요리 횟수
│              2025.12.01  │
└──────────────────────────┘
```

### 5.3 태그 추가 인터랙션

1. 카드의 `[+태그]` 버튼 터치
2. 팝오버 등장:
   - 기존 내 태그 목록 (클릭으로 즉시 부착)
   - 검색 인풋 (없는 태그는 엔터로 바로 생성)
   - 색상 선택 (8색 원형 버튼)
3. 팝오버 닫기 → 카드에 태그 즉시 반영

### 5.4 "만들어봤어요" 플로우

1. 레시피 상세 페이지 하단 버튼: `🍳 오늘 만들었어요`
2. 별점 입력 모달 (선택):
   ```
   이 레시피 어떠셨나요?
   ☆ ☆ ☆ ☆ ☆
   [나중에 입력할게요]  [저장]
   ```
3. 저장 시: `cooked_count` +1, `last_cooked_at` 업데이트, 별점 기록
4. 3회 달성 시 카드에 "단골 레시피 🏆" 뱃지 자동 표시

## 7. 유튜브 대비 개선 포인트

| 유튜브 저장 | 이 시스템 |
|------------|-----------|
| 재생목록 이름만 | 색상 태그 + 자동 카테고리 |
| 저장만 가능 | 만든 날짜, 별점, 횟수 기록 |

| 정렬 미지원 | 별점순, 최근 요리순 등 정렬 |
