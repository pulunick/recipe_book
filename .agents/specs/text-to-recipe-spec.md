# 텍스트 → 레시피 변환 기능 명세

작성일: 2026-03-05

---

## 개요

사용자가 자유형식의 텍스트(구어체, 메모, 블로그 글 등)를 입력하면, AI가 유튜브 분석과 동일한 구조화된 레시피 JSON으로 변환해주는 기능.

**차별점**: 유튜브 URL이 없는 레시피(할머니 손맛, 내 창작 레시피, 친구한테 받은 레시피 등)도 해먹당에 저장 가능.

---

## 사용 시나리오

### 시나리오 1: 카카오톡으로 받은 레시피

친구가 카카오톡으로 보내준 텍스트 레시피를 복붙해서 넣으면 → 깔끔한 레시피카드로 변환

### 시나리오 2: 내 창작 레시피

"일단 냉장고 열어서 신김치 꺼내셈. 스팸이나 참치 있으면 무조건 넣는게 맛있음..."
같은 구어체/반말 텍스트도 → 정형화된 레시피로 변환

### 시나리오 3: 블로그 레시피 복붙

블로그 레시피 본문 전체 복붙 → 불필요한 광고성 텍스트 제거 후 핵심만 추출

---

## 입력 스펙

- **형식**: 자유형식 한국어 텍스트 (구어체/문어체 무관)
- **최소 길이**: 50자 (너무 짧으면 레시피로 판단 불가)
- **최대 길이**: 5,000자
- **언어**: 한국어 우선 (향후 다국어 가능)

---

## 백엔드

### 새 엔드포인트

```
POST /extract-recipe-from-text
```

**Request:**
```json
{
  "text": "일단 냉장고 열어서 신김치 꺼내셈...",
  "title": "김치볶음밥"   // 선택 입력 (없으면 AI가 자동 생성)
}
```

**Response:** 기존 `Recipe` 스키마와 동일

### ai_engine.py 변경

기존 `analyze_recipe(audio_file, metadata)` 옆에 새 함수 추가:

```python
async def analyze_recipe_from_text(text: str, title: str | None) -> Recipe:
    ...
```

**프롬프트 전략:**
- 기존 오디오 분석 프롬프트 구조 그대로 재사용
- 오디오 파일 없이 텍스트만 입력
- "구어체/비표준 표현을 표준 레시피 형식으로 변환하라" 지침 추가
- `is_recipe` 체크 동일하게 적용 (레시피 아닌 텍스트 거부)

### DB 저장 방식

- `video_id`: `null` (유튜브 없음)
- `video_url`: `null`
- `source`: `"text"` 컬럼 추가 필요 (기존: `"youtube"`)
- 캐싱: 텍스트 입력은 캐시 없음 (매번 새로 분석)

---

## DB 변경

### recipes 테이블

```sql
ALTER TABLE recipes
ADD COLUMN source VARCHAR(20) DEFAULT 'youtube';
-- 값: 'youtube' | 'text'
```

`video_id` UNIQUE 제약 조건은 `NULL`에 대해 적용 안 되므로 텍스트 레시피 다수 저장 가능.

## 프론트엔드 UI

### 홈 페이지 입력 영역 변경

현재 URL 입력창 → 탭으로 전환:

```
[🎬 유튜브 URL] [📝 텍스트 직접 입력]  ← 탭
```

**유튜브 탭 (기존):**
```
┌─────────────────────────────────┐
│ https://youtube.com/...         │
└─────────────────────────────────┘
[레시피 정리하기]
```

**텍스트 탭 (신규):**
```
제목 (선택): [                    ]

┌─────────────────────────────────┐
│ 레시피를 자유롭게 적어주세요.    │
│                                 │
│ 구어체, 메모, 카카오톡으로 받은  │
│ 레시피 등 뭐든 괜찮아요.        │
│                                 │
│                                 │
└─────────────────────────────────┘
50자 이상 입력 시 활성화
[레시피로 정리하기]
```

### 플로우

```
텍스트 입력 → [레시피로 정리하기]
    → 로딩 (Gemini 분석 중)
    → 결과를 편집 모드로 바로 오픈
        (재료/단계/팁 전부 수정 가능 — 레시피 개인 편집과 동일한 UI)
    → [내 레시피북에 저장] 버튼 클릭 → 저장 완료
```

유튜브 레시피(자동 저장)와 달리 텍스트는 **항상 편집 모드 먼저**, 확인 후 수동 저장.

### 결과 표시

편집 모드로 표시 (레시피 개인 편집 UI 재사용).

### 썸네일 우선순위 (Fallback 체인)

```
1순위: 사용자가 직접 업로드한 이미지
2순위: 카테고리별 기본 이미지 (한식/양식/중식/일식 등)
3순위: 타이틀 텍스트 카드 (이미지 없이 레시피 제목만 표시)
```

**카테고리 기본 이미지**: 카테고리별 일러스트/사진을 `/static/category/` 에 준비.
**타이틀 카드**: 배경색(카테고리별 색상) + 흰 텍스트로 제목 표시. CSS만으로 구현.
