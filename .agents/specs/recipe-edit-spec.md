# 레시피 개인 편집 기능 명세

작성일: 2026-03-05

---

## 개요

AI가 추출한 원본 레시피를 유지하면서, 사용자가 자신만의 버전으로 재료·조리 단계를 수정할 수 있는 기능.

**핵심 원칙**: 원본(`recipes` 테이블)은 절대 수정하지 않는다. 수정본은 `user_collections`에 별도 저장.

---

## 사용 시나리오

1. 유튜브에서 추출한 김치볶음밥 레시피를 내 레시피북에 저장
2. 실제로 요리해보니 설탕이 너무 많음 → "설탕 1큰술" → "설탕 반큰술"로 수정
3. 간장을 화이트 와인 식초로 대체 → 재료 이름 수정
4. 조리 단계 중 "불 세기" 설명이 부족해서 내 경험을 추가
5. 저장 후 내 레시피북에서 수정된 버전으로 표시됨

## DB 설계

### user_collections 테이블 변경

```sql
ALTER TABLE user_collections
ADD COLUMN recipe_override JSONB DEFAULT NULL;
```

`recipe_override` 구조 (수정된 섹션 전체를 덮어씀):

```json
{
  "ingredients": [
    { "name": "신김치", "amount": "200g", "unit": "", "category": "주재료" },
    { "name": "스팸", "amount": "반통", "unit": "", "category": "주재료" },
    { "name": "설탕", "amount": "반큰술", "unit": "", "category": "양념", "note": "원본 1큰술에서 줄임" }
  ],
  "steps": [
    { "order": 1, "description": "수정된 단계 설명", "timer_minutes": 2, "note": "불 약하게 해야 안 탐" },
    { "order": 2, "description": "..." }
  ],
  "tip": "내가 직접 쓴 팁 (AI 팁 대체)"
}
```

**왜 전체 배열인가**: 추가/삭제/순서 변경을 모두 지원하려면 index 기반 부분 패치가 아닌 전체 배열 교체가 필요. `null`이면 원본 사용, 값이 있으면 전체 오버라이드.

## API 변경

### PATCH /collections/{id} 확장

기존에 `custom_tip`만 받던 것에 `recipe_override` 추가:

```json
{
  "custom_tip": "...",
  "recipe_override": {
    "ingredients": [...],
    "steps": [...],
    "tip": "..."
  }
}
```

### GET /collections/{user_id} 응답

기존 그대로 `recipe_override` 필드 포함해서 반환.
프론트에서 "원본 + override 머지" 처리.

## 프론트엔드 UI

### 진입점

`/my-recipes/[id]` 상세 페이지 상단에 **[레시피 수정]** 버튼 추가.

### 편집 모드

버튼 클릭 시 해당 페이지가 편집 모드로 전환 (별도 라우트 X, 인라인 전환):

```
[수정 중] 표시 배너
재료 항목마다 [✏️ 수량 수정] 버튼
단계 항목마다 [✏️ 단계 수정] 버튼
[변경 저장] / [취소] 버튼
```

### 재료 수정 UI

```
참기름    1큰술  →  [1큰술        ] [메모: 선택 입력         ]
설탕      1큰술  →  [반큰술       ] [메모: 너무 달아서 줄임   ]
```

### 단계 수정 UI

```
3단계: 팬에 기름을 두르고...
[textarea: 수정된 단계 설명 ]
[메모: 이 단계에서 주의할 점]
```

### 수정된 항목 표시

수정된 재료/단계는 원본과 다름을 시각적으로 표시:
- 좌측에 테라코타 선 강조
- "(수정됨)" 뱃지
- 원본 보기 토글 (접어두기)
