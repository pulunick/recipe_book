# 상황 태그 기능 명세 (Situational Tags)

## 개요

레시피에 상황/맥락 태그를 추가하여 "분위기 좋은 음식", "간편한 조리 음식" 같은 상황별 탐색을 지원한다.

## 배경

- 현재 카테고리(한식/양식 등)와 텍스트 검색만 지원
- "분위기 좋은 음식", "야식", "손님접대" 같은 상황적 개념 검색 불가
- 구글식 벡터 시맨틱 검색은 레시피 수 대비 과도한 인프라 → 태그 방식이 현실적
- Gemini가 **고정 목록에서 선택**하게 하면 주관적 개념도 일관성 있게 분류 가능

---

## 태그 목록 (확정 전 초안)

| 태그 | 설명 |
|------|------|
| `간편식` | 30분 이내, 특별한 기술 불필요 |
| `다이어트` | 저칼로리, 저지방, 샐러드 등 |
| `야식` | 늦은 시간 먹기 좋은 음식 |
| `손님접대` | 격식 있는 자리, 방문객 대접 |
| `특별한날` | 생일/기념일/데이트 등 분위기 있는 음식 |
| `해장` | 숙취 해소, 속 편한 음식 |
| `도시락` | 휴대 가능, 식어도 맛있는 음식 |
| `아이반찬` | 어린이 친화적, 자극 없는 음식 |
| `혼밥` | 1인분, 혼자 먹기 좋은 음식 |
| `술안주` | 주류와 잘 어울리는 음식 |
| `브런치` | 아침/점심 겸용, 카페 스타일 |
| `명절` | 설/추석 등 명절 음식 |

## 구현 계획

### 1. DB 변경
```sql
ALTER TABLE recipes ADD COLUMN situational_tags TEXT[] DEFAULT '{}';
CREATE INDEX idx_recipes_situational_tags ON recipes USING GIN(situational_tags);
```

### 2. AI 추출 프롬프트 수정 (ai_engine.py)
```
9. **Situational Tags (상황 태그)**:
   - 다음 고정 목록에서 해당하는 태그를 0~3개 선택하세요.
   - 목록: 간편식, 다이어트, 야식, 손님접대, 특별한날, 해장, 도시락, 아이반찬, 혼밥, 술안주, 브런치, 명절
   - 확실히 해당하는 것만 선택. 애매하면 제외.
   - 레시피가 아닌 경우 빈 배열 [].
```

### 3. API 수정 (main.py)
- `GET /recipes` 에 `tags` 필터 파라미터 추가
- `search_public_recipes` RPC에 태그 필터 조건 추가: `p_tags TEXT[]`

### 4. 프론트엔드 (탐색 탭)
- 카테고리 칩 아래 상황 태그 칩 행 추가 (또는 FilterBottomSheet에 통합)
- 빠른 접근용 상단 노출 태그: 간편식, 특별한날, 야식, 다이어트, 해장

---

## 기존 레시피 처리

- 신규 추출 레시피는 자동 태그 포함
- 기존 레시피(52개)는 `force_refresh` 로 재추출하거나, 별도 백필 스크립트 실행
- 백필 우선순위: collection_count 상위 레시피부터
