# Recipe AI 에이전트 운영 규칙 (Operational Rules)

이 프로젝트에서 에이전트(Antigravity)는 다음과 같은 규칙을 엄격히 준수합니다.

## 1. CUD 및 스키마 변경 시 필수 컨펌

다음에 해당하는 모든 작업은 **실행 전 반드시 사용자에게 명시적인 승인**을 얻어야 합니다.

- **데이터 변경 (CUD)**: `INSERT`, `UPDATE`, `DELETE` 등 테이블 데이터를 수정하는 작업
- **구조 변경 (DDL)**: `CREATE`, `ALTER`, `DROP` 등 테이블, 스키마, 함수, 트리거의 생성 및 수정

### 보고 절차

- 작업 전, 실행할 **SQL 구문 또는 Python 코드**를 보여줍니다.
- 해당 작업이 시스템에 미치는 영향(예: "레시피 테이블 생성", "추출된 데이터 DB 저장")을 설명합니다.
- 사용자의 명시적 승인 메시지가 있을 때만 도구를 호출합니다.

## 2. 보안 및 권한 관리

- `supabase-prod` 환경은 조회(Read-only) 용도로만 사용합니다.
- API Key 등 민감 정보는 `.env`로 관리하며 로그 노출을 방지합니다.

## 3. 언어 설정

- **언어**: 모든 응답, 문서화, 주석 및 설명은 한국어로 작성합니다.
- **예외**: 코드 변수명이나 기술적 용어 등은 영어를 병용할 수 있습니다.

## 4. 데이터 및 명명 규칙

- 테이블명은 `recipes`와 같이 직관적인 이름을 사용합니다.
- 필드명은 `schemas.py`의 정의(`title`, `summary`, `ingredients_markdown`, `steps_markdown`, `flavor`, `tip`, `video_url`)를 따릅니다.
- 맛 지표(`flavor`)는 `JSON` 타입으로 저장합니다.

## 5. AI 추론 원칙

- **근거 중심**: 오디오와 메타데이터에 없는 내용은 절대 임의로 생성하지 않습니다.
- **리포트 스타일**: NotebookLM 스타일의 풍부한 마크다운 리포트 형식을 유지합니다.
- **오디오 우선**: 자막보다 실제 음성 분석 결과를 우선합니다.
