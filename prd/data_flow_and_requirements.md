# 🍳 레시피 AI 에이전트: 데이터 흐름 및 요구사항 명세서 (PRD)

본 문서는 유튜브 요리 영상으로부터 정밀한 레시피 데이터를 추출하고, 이를 개인화된 데이터로 변환하는 '입맛 저격 레시피 AI' 시스템의 데이터 흐름과 상세 요구사항을 정의합니다.

---

## 1. 시스템 아키텍처 개요

시스템은 크게 4가지 레이어로 구성됩니다.

1.  **Frontend (Web)**: 사용자로부터 유튜브 URL을 입력받고, 분석된 구조화 데이터를 시각적으로 제공합니다.
2.  **API Server (FastAPI)**: 전반적인 분석 워크플로우를 제어하며, DB 조회 및 AI 엔진과의 통신을 담당합니다.
3.  **AI Analysis Engine (Gemini 2.5 Flash)**: 멀티모달(오디오+텍스트) 분석을 통해 비정형 영상 데이터를 정형화된 JSON 데이터로 변환합니다.
4.  **Database (Supabase)**: 원본 레시피(Canonical)와 사용자별 개인화 정보(Personal)를 분리하여 저장합니다.

---

## 2. 데이터 흐름 (Data Flow)

분석 요청부터 결과 반환까지의 단계별 흐름은 다음과 같습니다.

### Step 1: 요청 및 캐시 확인 (Request & Cache Check)

- 사용자가 프론트엔드에서 유튜브 URL을 입력하여 분석을 요청합니다.
- 백엔드는 Supabase의 `recipes` 테이블에서 해당 `video_url`이 이미 존재하는지 확인합니다.
- **Case A (이미 존재하는 경우)**: DB에서 데이터를 즉시 읽어와 사용자에게 반환하며, 아래의 무거운 분석 단계를 생략합니다. (성능 최적화 및 비용 절감)
- **Case B (새로운 영상인 경우)**: 다음의 분석 프로세스를 시작합니다.

### Step 2: 미디어 데이터 추출 (Media Extraction)

- `yt-dlp` 라이브러리를 사용하여 해당 유튜브 영상에서 오디오 트랙만 MP3 형식으로 추출합니다.
- 추출된 오디오 파일은 서버에 임시로 저장됩니다.

### Step 3: AI 기반 멀티모달 분석 (AI Analysis)

- 임시 저장된 오디오 파일을 Gemini 2.5 Flash 모델에 업로드합니다.
- **분석가 페르소나**: Gemini는 'AI 요리 데이터 분석가'로서 오디오의 대화 내용(정량, 팁, 뉘앙스)을 최우선으로 분석합니다.
- **구조화**: 단순 텍스트가 아닌, 정의된 규격에 맞춘 JSON 데이터를 생성합니다. (재료 카테고리화, 단계별 타이머 추출 등)

### Step 4: 데이터 검증 및 저장 (Validation & Persistence)

- AI가 생성한 JSON 데이터를 Pydantic 모델(`schemas.py`)을 통해 형식을 검증합니다.
- 검증된 깨끗한 데이터를 `recipes` 테이블에 저장합니다. 이때 `ingredients`, `steps`, `flavor`는 **JSONB** 타입으로 저장되어 나중에 개별 필드 조회가 가능합니다.

### Step 5: 결과 전시 (Presentation)

- 프론트엔드는 수신한 JSON 데이터를 바탕으로 재료(카테고리별), 조리법(순서 및 타이머), 맛 분석(레이더 차트 등)을 화면에 구성합니다.

---

## 3. 핵심 요구사항 (Key Requirements)

### 3.1 구조화된 레시피 데이터 (Structured Data)

- **재료 (Ingredients)**: 이름, 수량, 단위, 카테고리(주재료, 양념 등)가 분리된 객체 배열 형태여야 함.
- **조리 과정 (Steps)**: 단계 번호, 상세 설명, 타이머(옵션)를 포함해야 함.
- **맛 분석 (Flavor)**: 짠맛, 단맛, 매운맛, 신맛, 기름짐의 5대 지표를 1~5점 척도로 수치화해야 함.

### 3.2 원본과 개인화의 분리 (Data Separation)

- **Canonical Recipe (`recipes` 테이블)**: 모든 사용자에게 공통으로 제공되는 원본 데이터. 특정 사용자 ID에 종속되지 않음.
- **Personal Collection (`user_collections` 테이블)**: 사용자가 레시피를 저장하거나 수정할 때 생성됨. 원본 레시피 ID를 참조하며, 사용자의 개별 메모(팁)와 재료 가감 정보(`ingredient_adjustments`)를 담음.

### 3.3 운영 및 보안 규칙

- **언어 통일**: 모든 내부 주석, 로그 메시지, 사용자 노출 텍스트는 **한국어**를 원칙으로 함.
- **성능**: 동일 영상에 대한 중복 분석을 방지하기 위해 URL 기반의 캐싱 로직을 유지해야 함.
- **유연성**: 재료의 수량이나 단위가 명확하지 않은 경우에도 시스템 에러 없이 빈 문자열 등으로 처리하여 서비스 가용성을 확보함.

---

## 4. 향후 확장 요구사항 (Roadmap)

- **개인 프로필 매칭**: 사용자의 선호 맛 지표와 분석된 레시피 지표를 비교하여 "이 레시피는 사용자님의 평소 취향보다 더 맵습니다"와 같은 경고/조정 메시지 제공.
- **냉장고 연동**: 추출된 재료 리스트를 바탕으로 현재 사용자의 냉장고에 없는 재료를 자동으로 체크하여 쇼핑 리스트 생성.
- **영상 클립 연동**: 각 조리 단계별로 해당 장면이 나오는 영상의 타임스탬프를 함께 저장하여 이동 기능 제공.
